# E-commerce Website Automation Testing Challenge

**Difficulty:** Senior  
**Deadline:** 3 days

## Scenario

An e-commerce website allows users to browse products, add them to cart, and checkout using various payment methods. The website has both desktop and mobile versions, and it uses a complex user authentication system. The task is to create a comprehensive test plan and automate critical user journeys using Selenium or Cypress.

## Requirements

1. Create a test plan document that covers user authentication, product browsing, adding to cart, and checkout functionality
2. Implement at least one automation script using Selenium or Cypress to cover the critical user journeys
3. Ensure the test plan and automation script cover both desktop and mobile versions of the website
4. Use a data-driven approach to test different payment methods and user scenarios

## Deliverables

- A test plan document + at least one automation script (any framework)

## Evaluation Criteria

- Comprehensive coverage of critical user journeys in the test plan and automation script
- Effectiveness of the automation script in covering different payment methods and user scenarios
- Quality of the test plan document, including clarity, organization, and attention to detail
- Ability to adapt to complex user authentication systems and multiple website versions
